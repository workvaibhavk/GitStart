—— 01 GIT BASICS

Core Concepts Explained Simply

Everything you need to understand before writing your first commit.Clear, no-fluff explanations with real commands.

🗃️

01 —— Concept

Repository

A repository (repo) is a folder tracked by Git. It stores your project's entire history — every change you've ever made.

$ git init

📸

02 —— Concept

Commit

A commit is a saved snapshot of your files at a specific moment. Think of it like a checkpoint in a video game you can always return to.

$ git commit -m "message"

🌿

03 —— Concept

Branch

A branch is an independent line of development. Create one to work on a feature without affecting the main stable code.

$ git checkout -b feature

🔀

04 —— Concept

Merge

Merging combines changes from one branch into another. When your feature is ready, merge it back into the main branch.

$ git merge feature

☁️

05 —— Concept

Push & Pull

Push sends your local commits to GitHub. Pull downloads changes from GitHub to your local machine. The sync cycle.

$ git push / git pull

🔱

06 —— Concept

Fork

Forking creates your own copy of someone else's repository. The gateway to open-source contribution — change freely, then propose your changes.

GitHub: Fork button → Clone

—— 02 Contribution Workflow

Your First PR Step by Step

The exact workflow you'll use to contribute to any open-source project on GitHub. Follow these 8 steps every time.

01

Find a Project

Search GitHub for beginner-friendly repos with the "good first issue" label

github.com/explore

02

Fork the Repo

Click Fork to create your own copy of the repository under your GitHub account

GitHub → Fork button

03

Clone Locally

Download the forked repository to your computer so you can edit it

$ git clone

04

Create Branch

Never work directly on main. Create a descriptive branch for your changes

$ git checkout -b fix

05

Make Changes

Edit files, fix the bug, improve docs, or add your feature

Edit your files

06

Commit Changes

Stage and commit your work with a clear, descriptive commit message

$ git commit -m "fix"

07

Push Branch

Send your local branch to your forked repository on GitHub

$ git push origin fix

08

Open Pull Request

On GitHub, open a PR to propose merging your changes into the original repo

GitHub → New PR 🎉
